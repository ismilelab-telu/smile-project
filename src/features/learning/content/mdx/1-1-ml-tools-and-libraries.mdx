Setelah mengenal alur kerja ML, sekarang kita masuk ke alat kerja. Bagian ini bukan daftar nama untuk dihafal. Tujuannya lebih praktis: kamu tahu **alat mana dipakai untuk pekerjaan apa**.

Dalam proyek ML, library membantu kita:

- memuat dan memeriksa data;
- menghitung nilai numerik;
- membuat visualisasi;
- melatih model;
- mengevaluasi hasil.

Library membuat kita tidak perlu membangun semuanya dari nol. Energi belajar bisa dipakai untuk hal yang lebih penting: memahami masalah, membaca data, dan mengambil keputusan modeling yang masuk akal.

## Python sebagai Bahasa Kerja

**[Python](https://www.python.org/)** populer di machine learning karena mudah dibaca dan ekosistem library-nya besar. Satu bahasa bisa dipakai untuk eksplorasi data, eksperimen model, sampai evaluasi.

Python sering dipakai lewat dua cara:

- **Interactive mode** untuk mencoba potongan kecil dan langsung melihat hasil.
- **Script mode** untuk kode yang lebih rapi, panjang, dan bisa dijalankan ulang.

Dalam praktik, keduanya saling melengkapi. Kamu bisa bereksperimen secara interaktif, lalu merapikan langkah yang sudah jelas menjadi script atau notebook yang bisa diulang.

## Notebook untuk Eksperimen

**Notebook** adalah tempat yang nyaman untuk belajar dan bereksperimen. Di satu tempat, kamu bisa menulis kode, mencatat alasan, melihat tabel, dan menampilkan grafik.

Tool yang sering dipakai:

- **[Jupyter Notebook](https://jupyter.org/)** untuk eksperimen lokal atau berbasis browser.
- **[Google Colab](https://colab.research.google.com/)** untuk mulai cepat tanpa banyak setup lokal.
- **[IBM Watson Studio](https://www.ibm.com/products/watson-studio)** untuk lingkungan cloud yang lebih terkelola.

Notebook tetap perlu disiplin. Kalau cell dijalankan lompat-lompat dan catatan tidak jelas, hasil eksperimen sulit dipercaya.

Pegangan notebook yang sehat:

- urutan cell jelas;
- output bisa diulang;
- catatan menjelaskan alasan langkah;
- eksperimen tidak bergantung pada state lama yang tersembunyi.

## Library untuk Data

Tahap awal ML biasanya bukan model, tetapi **memahami dataset**. Di sini, **[NumPy](https://numpy.org/)** dan **[Pandas](https://pandas.pydata.org/)** sering muncul.

**NumPy** membantu komputasi numerik:

- array;
- vektor;
- matriks;
- operasi matematika;
- perhitungan yang efisien.

**Pandas** membantu data berbentuk tabel:

- membaca file seperti `.csv`;
- melihat sampel baris;
- memeriksa nama kolom;
- mengecek tipe data;
- mencari _missing value_;
- memfilter baris;
- membuat ringkasan sederhana.

Pada tahap ini, kita memeriksa **schema** dataset: nama kolom, tipe data, dan arti tiap kolom.

## Library untuk Visualisasi

Tabel bisa memberi angka, tetapi grafik sering membuat pola lebih cepat terlihat.

**[Matplotlib](https://matplotlib.org/)** cocok untuk grafik dasar:

- line chart;
- bar plot;
- histogram;
- scatter plot.

Matplotlib memberi kontrol besar atas detail grafik, sehingga cocok saat kamu ingin mengatur tampilan visual satu per satu.

**[Seaborn](https://seaborn.pydata.org/)** dibangun di atas Matplotlib. Keunggulannya adalah membuat visualisasi statistik lebih cepat dan rapi, terutama saat bekerja dengan data tabular seperti Pandas DataFrame.

- distribusi data;
- hubungan antar variabel;
- heatmap;
- box plot;
- pola kategori.

## Contoh Sederhana

Misalnya kita ingin melihat hubungan `x` dan `y` dari dataframe `df`.

Dengan Matplotlib, kita mengatur plot secara eksplisit:

```python
import matplotlib.pyplot as plt

plt.scatter(df["x"], df["y"])
plt.xlabel("X")
plt.ylabel("Y")
plt.show()
```

Dengan Seaborn, plot statistik yang sama bisa ditulis lebih ringkas:

```python
import seaborn as sns

sns.scatterplot(data=df, x="x", y="y")
```

Intinya:

- **Matplotlib** memberi kontrol penuh.
- **Seaborn** lebih cepat untuk membuat plot statistik yang rapi.

Visualisasi awal membantu melihat:

- sebaran nilai;
- perbandingan kategori;
- hubungan sederhana antar kolom;
- nilai yang tampak tidak biasa;
- ringkasan visual yang cepat dibaca.

Untuk lesson ini, cukup pahami bahwa library visualisasi membantu kita membaca data sebelum masuk ke tahap berikutnya.

## Library untuk Machine Learning

**[Scikit-learn](https://scikit-learn.org/)** sering menjadi pilihan awal untuk belajar ML klasik. Library ini menyediakan:

- klasifikasi;
- regresi;
- clustering;
- preprocessing;
- pipeline;
- cross-validation;
- metrik evaluasi.

Dalam banyak proyek, kita mulai dari **baseline model**. Baseline adalah model sederhana yang menjadi pembanding awal.

Baseline berguna karena:

- memberi skor pembanding;
- menunjukkan apakah model kompleks benar-benar membantu;
- membuat eksperimen lebih mudah dinilai;
- mencegah kita langsung memilih model besar tanpa alasan.

Untuk deep learning, ada library lain:

- **[TensorFlow](https://www.tensorflow.org/)** banyak dipakai di industri.
- **[PyTorch](https://pytorch.org/)** populer untuk riset dan eksperimen.
- **[Keras](https://keras.io/)** memberi antarmuka yang lebih sederhana untuk prototipe.

Deep learning tidak otomatis menjadi pilihan pertama. Jika targetnya angka, seperti waktu pengiriman, model regresi sederhana di Scikit-learn sering menjadi awal yang lebih jernih.

## Contoh Alur Praktis

Bayangkan kamu mendapat file `.csv` food delivery dan diminta membuat model awal prediksi waktu pengiriman.

Alur yang sehat:

1. Mulai di notebook agar asumsi, kode, dan output bisa dilacak.
2. Gunakan **Pandas** untuk membaca data.
3. Lihat sampel baris untuk memahami bentuk dataset.
4. Periksa **schema**: nama kolom, tipe data, dan arti kolom.
5. Cek _missing value_ dan nilai yang terlihat aneh.
6. Buat visualisasi awal dengan ringkasan dan grafik sederhana.
7. Plot **target**, yaitu waktu pengiriman dalam menit.
8. Periksa **fitur** seperti jarak, cuaca, trafik, waktu hari, kendaraan, waktu persiapan, dan pengalaman kurir.
9. Latih **baseline** dengan **Scikit-learn**.
10. Pertimbangkan model lebih kompleks hanya jika baseline belum cukup.
