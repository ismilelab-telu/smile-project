In the previous lesson, you chose and validated a dataset source. The link and dataset description give context, but the data cannot be analyzed until it actually enters the project.

Collected data should not stop as a source note. The next step is **data loading**: reading the dataset from its source into a working environment, such as a Python notebook, so the data can be inspected as a table.

## What Data Loading Means

**Data loading** is the process of taking data from a file, database, API, or another source and putting it into a data structure the machine learning project can use.

For tabular data, the most common structure in Python is a **Pandas DataFrame**. A DataFrame makes data easier to inspect:

- rows represent examples or records;
- columns represent fields about each example;
- data types, missing values, and first summaries can be checked quickly.

At this stage we are not cleaning data yet and not training a model yet. The goal is to make sure the chosen data enters correctly before cleaning, EDA, feature preparation, or modeling.

## Why This Matters

Data loading is the entry point to the data workflow. If the data fails to load or is read incorrectly, every later stage becomes fragile.

There are three main reasons:

- **First step**: without successfully loaded data, the model has no examples to learn from or be tested on.
- **Data integrity**: loading helps check whether columns, data types, rows, and missing values were read as expected.
- **Readiness for the next process**: data loaded into the right structure is easier to explore, clean, transform, and use for model experiments.

For the food delivery case, data loading answers a simple question: did the dataset file really become an order table we can audit?

## Common Sources and Formats

**Pandas** can read many data formats. You do not need to memorize all of them, but you should know that the loading method depends on the source.

Common formats include:

- **CSV**: a common tabular format with the `.csv` extension, usually read with `pd.read_csv("file.csv")`.
- **Excel**: spreadsheet files with `.xls` or `.xlsx` extensions, usually read with `pd.read_excel("file.xlsx")`.
- **JSON**: object-based data with the `.json` extension, usually read with `pd.read_json("file.json")`.
- **HTML table**: tables from an `.html` page or file, usually read with `pd.read_html("file.html")`.
- **SQL database**: tables from a database, usually read through a query such as `pd.read_sql_query("SELECT * FROM orders", connection)`.
- **Parquet**: an efficient columnar format for larger data with the `.parquet` extension, usually read with `pd.read_parquet("file.parquet")`.

Pandas also supports other formats such as `feather`, `hdf5`, `stata`, `sas`, `spss`, `pickle`, `orc`, and clipboard. In early practice, CSV and Excel are the most common. SQL or Parquet often appears when data is already more structured in production systems.

Simple example:

```python
import pandas as pd

df = pd.read_csv("food_delivery.csv")
```

After that code runs, `df` becomes a DataFrame that can be checked before it is processed further.

## After the Data Enters

Do not train a model immediately after opening a file. First make sure the loaded data matches expectations.

Check the first signals:

- a few first rows with `df.head()`;
- table size with `df.shape`;
- column names and expected fields;
- data types with `df.dtypes`;
- missing values with a blank-value check.

If the dataset comes from Kaggle, the file often arrives as a ZIP. Download the ZIP from the dataset page, extract it, then choose the main tabular file such as `.csv` before reading it with Pandas.

After the first checks look safe, continue to cleaning, EDA, or feature preparation.
