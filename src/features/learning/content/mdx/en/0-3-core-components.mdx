Machine learning is not only choosing an algorithm. A tidy ML project always starts with a clear problem, then moves through data, model, training, prediction, and evaluation.

If one of these parts is vague, the final result becomes vague too. A sophisticated model can still miss the point if the target is unclear, the data is irrelevant, or the evaluation is not honest.

## Main Components

1. **Problem or task** explains what the model should help with. Examples include predicting demand, grouping customers, or detecting risky transactions.
2. **Data** provides examples the model can learn from. Data can be tables, text, images, audio, activity logs, or other forms.
3. **Model** is a mathematical system that tries to capture patterns from data. A model receives input and produces output such as a number, category, score, or group.
4. **Training** is the process of adjusting the model using data. In this stage, the model changes its internal values so the output becomes closer to the learning target.
5. **Prediction or inference** is the process of using a trained model on new examples.
6. **Evaluation** judges whether the model is actually useful on data or situations that were not used to train it.

## Big Workflow

For the foundation level, keep this workflow in mind:

1. understand the problem;
2. prepare the data;
3. choose an initial model;
4. train the model;
5. use the model for prediction;
6. evaluate the result.

Other components such as feature engineering, hyperparameter tuning, and deployment are also important. However, they are easier to understand after this basic workflow is clear.

The key point is that the model is only one part of the ML system. A good project needs the right problem, reasonable data, a controlled learning process, and evaluation that can be trusted.
