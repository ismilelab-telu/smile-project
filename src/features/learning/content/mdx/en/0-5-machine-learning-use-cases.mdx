Machine learning is strong when a problem has patterns that can be learned from examples. That is why the first question before using ML is not "what is the coolest model?", but "is there a pattern from past data that can help decisions for new examples?"

ML use cases usually involve prediction, grouping, recommendation, risk detection, or prioritization. All of these use cases need data and a clear way to evaluate the result.

## Suitable Use Cases

ML tends to make sense when:

- enough historical data or examples exist;
- the pattern is hard to write as a complete manual rule;
- decisions need to be made repeatedly for many cases;
- the model output quality can be checked;
- model mistakes can be understood and managed.

Examples of suitable use cases:

- predicting food delivery time from distance, weather, traffic, and vehicle type;
- grouping customers based on purchase patterns;
- detecting risky transactions from previous transaction patterns;
- recommending products from interaction history;
- classifying email as spam or not spam.

## Use Cases That Do Not Need ML

Not every problem needs machine learning. If a problem can be solved with a simple stable rule, ML can be unnecessary.

Examples:

- calculating checkout total with price multiplied by quantity;
- calculating tax from a fixed formula;
- turning on an alarm at the same time every day;
- simple format validation, such as checking that an email has an `@` sign;
- giving a fixed 10 percent discount to every customer.

In examples like these, ordinary rules are clearer, easier to test, and easier to explain.

## Initial Questions

Before using ML, check three things: whether relevant data exists, whether the pattern needs to be learned from examples, and whether the result can be evaluated. If the answers are not clear, the problem needs to be cleaned up before moving into modeling.
