Types of machine learning are separated by two things: **what kind of answer we want** and **what kind of feedback is available**.

This matters because the problem type determines the data shape, the models that make sense, and the way we evaluate results. If we misread the problem type, a model can look like it works while answering the wrong question.

## Supervised Learning

**Supervised learning** uses examples that already have answers. The model sees inputs and targets, then learns the relationship between them.

Two common forms:

- **Regression** predicts numbers, such as delivery time in minutes or operating cost.
- **Classification** predicts categories, such as spam or not spam, low risk or high risk.

Supervised learning fits when you have historical data with **features** and a **target**.

## Unsupervised Learning

**Unsupervised learning** uses data without target answers. The goal is not to guess a provided label, but to find structure in the data.

The easiest example to remember is **clustering**. The model tries to group similar examples, such as customers with similar purchase patterns.

## Reinforcement Learning

**Reinforcement learning** trains an agent to choose actions based on rewards. The agent tries an action, receives feedback, then learns a strategy that produces better results.

You often see this idea in games, robotics, control systems, or decision simulations. For the foundation level, just remember that the feedback is not a direct answer target, but a reward from an action.

## Quick Way to Tell Them Apart

Look at the output and feedback source:

- numeric output usually points to **regression**;
- category output usually points to **classification**;
- groups without a target usually point to **clustering**;
- actions scored by rewards point to **reinforcement learning**.

This map is not for memorizing names. It helps you choose an initial direction before going into algorithms.
