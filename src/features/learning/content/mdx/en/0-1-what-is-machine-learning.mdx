Machine learning is a way to make computers **learn patterns from data**. Not because the computer becomes "smart" by magic, but because we give it examples, a goal, and a way to judge the result.

Arthur Samuel introduced the term machine learning in 1959:

> "A field of study that gives computers the ability to learn without being explicitly programmed."

That sentence sounds big, but the idea is simple: we do not need to write a rule for every condition. We can provide data, then let a model find patterns that help produce predictions, recommendations, scores, or decisions for new examples.

## From Rules to Patterns

Manual rules work well when the situation is clear. For example:

- if the distance is under 2 km, estimate delivery at 15 minutes;
- if the checkout total is above a threshold, give a discount;
- if an email format has no `@`, show an error.

The problem appears when the real world has too many conditions. Food delivery is not affected only by distance. Weather, traffic, time of day, vehicle type, preparation time, courier experience, and many small variations can all matter.

This is where machine learning becomes useful. A model looks at many previous deliveries, then learns that long routes with high traffic usually need more time than short routes on a quiet morning.

## When ML Makes Sense

Machine learning is usually worth trying when:

- a pattern exists, but it is hard to write as a complete rule;
- the pattern can change over time;
- decisions need to be repeated across many examples;
- relevant data is available;
- the model result can be checked with a clear measure.

If the problem is only a fixed formula, ML can be unnecessary. Calculating checkout total from price and quantity is better handled with ordinary rules.

## Stay Careful

A model does not understand the world like a human. It only learns from the data, target, and feedback that are available.

That means:

- poor data can produce a poor model;
- a wrong target makes the model learn the wrong thing;
- a new context can make old patterns stop fitting.

The main idea: **machine learning learns patterns from data**, instead of running a fixed list of rules for every condition.
