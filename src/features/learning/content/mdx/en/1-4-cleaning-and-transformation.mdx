Cleaning and transformation prepare data so it can be analyzed and modeled more safely. The two are related, but they answer different questions.

Cleaning asks whether the data is trustworthy enough to use. Transformation asks whether the data representation is useful enough for analysis or modeling.

## Cleaning

Cleaning handles data quality issues. Examples include:

- missing values;
- duplicate records;
- inconsistent labels such as `sunny`, `Sunny`, and `clear weather`;
- impossible values such as negative demand;
- invalid dates;
- category values that do not match the expected list;
- target values that are empty or cannot be verified.

Cleaning should be done for a reason. Removing rows, filling missing values, or changing labels can change the patterns learned by the model.

## Transformation

Transformation changes how information is represented. The goal can be making raw data easier to analyze or easier for a model to use.

Examples:

- turning timestamps into day of week or hour;
- standardizing category names;
- scaling numeric values;
- grouping rare categories;
- creating derived features from information already known before prediction.

Transformation is not automatically good. A new feature is useful only if it is meaningful, available at prediction time, and not a shortcut to the answer.

## Watch for Leakage

The main risk is target leakage. Leakage happens when an input feature contains information that will not be known when the model is used in the real world.

For example, if the model predicts drinks sold before a shift starts, final shift revenue is not a safe feature. That column is only known after the shift and is very close to the target.

Good cleaning and transformation make the dataset clearer without slipping future information into the model inputs.
