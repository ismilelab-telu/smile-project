After learning the ML workflow, we can look at the tools. This lesson is not a list of names to memorize. The practical goal is: you know **which tool is used for which job**.

In an ML project, libraries help us:

- load and inspect data;
- compute numerical values;
- create visualizations;
- train models;
- evaluate results.

Libraries keep us from building everything from scratch. That leaves more learning energy for the important parts: understanding the problem, reading the data, and making reasonable modeling decisions.

## Python as a Working Language

**[Python](https://www.python.org/)** is popular in machine learning because it is readable and has a large library ecosystem. The same language can support data exploration, model experiments, and evaluation.

Python is often used in two ways:

- **Interactive mode** for trying small pieces and seeing results quickly.
- **Script mode** for cleaner, longer code that can be rerun.

In practice, both modes work together. You can explore interactively, then organize clearer steps into a script or notebook that can be repeated.

## Notebooks for Experiments

A **notebook** is a comfortable place to learn and experiment. In one place, you can write code, record reasons, inspect tables, and show charts.

Common tools:

- **[Jupyter Notebook](https://jupyter.org/)** for local or browser-based experiments.
- **[Google Colab](https://colab.research.google.com/)** for starting quickly without much local setup.
- **[IBM Watson Studio](https://www.ibm.com/products/watson-studio)** for a more managed cloud environment.

Notebooks still need discipline. If cells run out of order and notes are unclear, experiment results become hard to trust.

Healthy notebook habits:

- cell order is clear;
- outputs are reproducible;
- notes explain the reason for each step;
- experiments do not depend on hidden old state.

## Data Libraries

Early ML work usually starts not with a model, but with **understanding the dataset**. This is where **[NumPy](https://numpy.org/)** and **[Pandas](https://pandas.pydata.org/)** often appear.

**NumPy** helps with numerical computing:

- arrays;
- vectors;
- matrices;
- mathematical operations;
- efficient calculations.

**Pandas** helps with tabular data:

- reading files such as `.csv`;
- inspecting sample rows;
- checking column names;
- inspecting data types;
- finding _missing values_;
- filtering rows;
- creating simple summaries.

At this stage, we inspect the dataset **schema**: column names, data types, and what each column means.

## Visualization Libraries

Tables give numbers, but charts often make patterns easier to see.

**[Matplotlib](https://matplotlib.org/)** works well for basic charts:

- line charts;
- bar plots;
- histograms;
- scatter plots.

Matplotlib gives you detailed control over a chart, so it fits when you want to adjust visual elements one by one.

**[Seaborn](https://seaborn.pydata.org/)** is built on top of Matplotlib. Its advantage is making statistical visualization faster and cleaner, especially when working with tabular data such as Pandas DataFrames.

- data distributions;
- relationships between variables;
- heatmaps;
- box plots;
- category patterns.

## Simple Example

Suppose we want to see the relationship between `x` and `y` from a dataframe named `df`.

With Matplotlib, we set up the plot explicitly:

```python
import matplotlib.pyplot as plt

plt.scatter(df["x"], df["y"])
plt.xlabel("X")
plt.ylabel("Y")
plt.show()
```

With Seaborn, the same statistical plot can be written more compactly:

```python
import seaborn as sns

sns.scatterplot(data=df, x="x", y="y")
```

The point:

- **Matplotlib** gives full control.
- **Seaborn** is faster for clean statistical plots.

Early visualization helps reveal:

- value distributions;
- category comparisons;
- simple relationships between columns;
- values that look unusual;
- quick visual summaries.

For this lesson, it is enough to understand that visualization libraries help us read data before moving to the next stage.

## Machine Learning Libraries

**[Scikit-learn](https://scikit-learn.org/)** is often the first choice for learning classic ML. It provides:

- classification;
- regression;
- clustering;
- preprocessing;
- pipelines;
- cross-validation;
- evaluation metrics.

In many projects, we start with a **baseline model**. A baseline is a simple model used as an initial comparison.

A baseline helps because it:

- gives a comparison score;
- shows whether a complex model actually helps;
- makes experiments easier to judge;
- prevents us from choosing a large model without a reason.

For deep learning, other libraries appear:

- **[TensorFlow](https://www.tensorflow.org/)** is widely used in industry.
- **[PyTorch](https://pytorch.org/)** is popular for research and experimentation.
- **[Keras](https://keras.io/)** gives a simpler interface for prototyping.

Deep learning is not automatically the first choice. If the target is a number, such as delivery time, a simple regression model in Scikit-learn is often a clearer starting point.

## Practical Workflow Example

Imagine you receive a food delivery `.csv` and need to build an initial model for delivery-time prediction.

A healthy workflow:

1. Start in a notebook so assumptions, code, and outputs can be traced.
2. Use **Pandas** to read the data.
3. Inspect sample rows to understand the dataset shape.
4. Check the **schema**: column names, data types, and column meanings.
5. Check _missing values_ and unusual values.
6. Make early visualizations with summaries and simple charts.
7. Plot the **target**, which is delivery time in minutes.
8. Inspect **features** such as distance, weather, traffic, time of day, vehicle type, preparation time, and courier experience.
9. Train a **baseline** with **Scikit-learn**.
10. Consider a more complex model only if the baseline is not enough.
