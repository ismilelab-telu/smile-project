Pembagian data menjaga evaluasi agar tidak terlalu optimistis. Model bisa terlihat kuat pada contoh yang sudah pernah dilihat, tetapi pertanyaan sebenarnya adalah bagaimana model bekerja pada contoh baru.

Pembagian data memberi peran berbeda pada bagian dataset. Sebagian data dipakai untuk belajar, dan sebagian data ditahan untuk memperkirakan performa di masa depan.

## Latih, Validasi, dan Uji

Data latih dipakai untuk fit transformasi dan melatih model. Di sinilah model belajar pola.

Data validasi bisa dipakai untuk membandingkan pilihan selama pengembangan, seperti jenis model, pilihan fitur, atau hyperparameter.

Data uji disimpan untuk estimasi akhir setelah keputusan besar dibuat. Data uji sebaiknya berperan seperti data masa depan yang belum pernah dilihat, bukan papan skor yang dicek setiap perubahan kecil.

## Split Sebelum Belajar

Pembagian data sebaiknya dilakukan sebelum proses fitting apa pun yang belajar dari data. Ini mencakup model dan banyak langkah preprocessing.

Jika transformasi belajar dari semua data sebelum dibagi, informasi dari data yang seharusnya ditahan bisa bocor ke data latih. Evaluasi bisa terlihat lebih baik daripada performa nyata.

Alur kerja yang lebih aman:

1. bagi data;
2. fit transformasi pada data latih;
3. terapkan transformasi ke data validasi atau data uji;
4. latih model pada data latih;
5. evaluasi pada data yang ditahan.

## Mewakili Masalah Nyata

Pembagian data sebaiknya sesuai dengan cara model akan dipakai. Jika model memprediksi waktu pengiriman untuk order baru, split perlu memastikan variasi jarak, cuaca, trafik, dan kendaraan tetap terwakili.

Ide dasarnya sederhana: sisihkan beberapa contoh agar evaluasi terasa lebih dekat dengan dunia nyata.
