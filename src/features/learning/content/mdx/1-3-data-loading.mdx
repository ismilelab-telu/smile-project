Di lesson sebelumnya, kamu sudah memilih dan memvalidasi sumber dataset. Link dan deskripsi dataset memberi konteks, tetapi data belum bisa dianalisis sampai benar-benar masuk ke project.

Data yang sudah dikumpulkan jangan berhenti sebagai catatan sumber. Langkah berikutnya adalah **data loading**: membaca dataset dari sumbernya ke lingkungan kerja, misalnya notebook Python, agar datanya bisa diperiksa sebagai tabel.

## Apa Itu Data Loading

**Data loading** adalah proses mengambil data dari file, database, API, atau sumber lain, lalu memasukkannya ke struktur data yang bisa dipakai dalam project machine learning.

Untuk data tabular, struktur yang paling sering dipakai di Python adalah **Pandas DataFrame**. DataFrame membuat data lebih mudah dibaca:

- baris mewakili contoh atau record;
- kolom mewakili field tentang tiap contoh;
- tipe data, nilai kosong, dan ringkasan awal bisa diperiksa dengan cepat.

Pada tahap ini kita belum membersihkan data dan belum melatih model. Tujuannya adalah memastikan data yang dipilih bisa masuk dengan benar sebelum lanjut ke cleaning, EDA, feature preparation, atau modeling.

## Kenapa Ini Penting

Data loading adalah pintu masuk ke alur kerja data. Jika data gagal dimuat atau salah terbaca, tahap setelahnya ikut rapuh.

Ada tiga alasan utama:

- **Langkah awal**: tanpa data yang berhasil dimuat, model tidak punya contoh untuk belajar atau diuji.
- **Integritas data**: proses loading membantu mengecek apakah kolom, tipe data, baris, dan nilai kosong terbaca sesuai ekspektasi.
- **Kesiapan proses berikutnya**: data yang sudah masuk ke struktur yang tepat lebih mudah dieksplorasi, dibersihkan, ditransformasi, dan dipakai untuk eksperimen model.

Untuk kasus food delivery, data loading menjawab pertanyaan sederhana: apakah file dataset benar-benar terbaca sebagai tabel order yang bisa kita audit?

## Sumber dan Format Umum

**Pandas** bisa membaca banyak format data. Tidak perlu menghafal semuanya, tetapi kamu perlu tahu bahwa cara membaca data bergantung pada sumbernya.

Format yang sering muncul:

- **CSV**: format tabular umum dengan ekstensi `.csv`, biasanya dibaca dengan `pd.read_csv("file.csv")`.
- **Excel**: spreadsheet dengan ekstensi `.xls` atau `.xlsx`, biasanya dibaca dengan `pd.read_excel("file.xlsx")`.
- **JSON**: data berbasis objek dengan ekstensi `.json`, biasanya dibaca dengan `pd.read_json("file.json")`.
- **HTML table**: tabel dari halaman atau file `.html`, biasanya dibaca dengan `pd.read_html("file.html")`.
- **SQL database**: tabel dari database, biasanya dibaca lewat query seperti `pd.read_sql_query("SELECT * FROM orders", connection)`.
- **Parquet**: format kolumnar efisien untuk data besar dengan ekstensi `.parquet`, biasanya dibaca dengan `pd.read_parquet("file.parquet")`.

Pandas juga mendukung format lain seperti `feather`, `hdf5`, `stata`, `sas`, `spss`, `pickle`, `orc`, dan clipboard. Dalam praktik awal, CSV dan Excel paling sering ditemui, lalu SQL atau Parquet muncul ketika data sudah lebih terstruktur di sistem produksi.

Contoh sederhana:

```python
import pandas as pd

df = pd.read_csv("food_delivery.csv")
```

Setelah kode itu berjalan, `df` menjadi DataFrame yang bisa dicek sebelum diolah lebih jauh.

## Setelah Data Masuk

Jangan langsung melatih model setelah membuka file. Pastikan dulu data yang dimuat sesuai ekspektasi.

Cek hal awal:

- beberapa baris pertama dengan `df.head()`;
- ukuran tabel dengan `df.shape`;
- nama kolom dan field yang diharapkan;
- tipe data dengan `df.dtypes`;
- missing values dengan pengecekan nilai kosong.

Jika dataset berasal dari Kaggle, file sering berbentuk ZIP. Unduh ZIP dari halaman dataset, extract isinya, lalu pilih file tabular utama seperti `.csv` sebelum dibaca dengan Pandas.

Setelah cek awal aman, baru lanjut ke cleaning, EDA, atau persiapan fitur.
