Memuat data berarti membawa sumber data mentah ke bentuk yang bisa diperiksa project. Sumbernya bisa berupa file CSV, spreadsheet, tabel database, respons API, export log, atau query dari data warehouse.

Memuat data belum masuk ke pemodelan. Ini adalah tahap untuk memastikan data masuk dengan benar sebelum pembersihan, analisis, persiapan fitur, atau pelatihan.

## Output dari Proses Memuat Data

Untuk pekerjaan ML tabular, proses memuat data biasanya menghasilkan tabel atau dataframe. Baris harus mewakili contoh yang ingin dianalisis, sedangkan kolom harus mewakili field tentang tiap contoh.

Jika bentuk tabel salah, semua tahap setelahnya menjadi kurang bisa dipercaya. Model tetap bisa dilatih pada tabel yang salah bentuk, tetapi hasilnya belum tentu bermakna.

## Pengecekan Pertama

Jangan langsung melatih model setelah membuka file. Pastikan dulu data yang dimuat sesuai ekspektasi.

Cek:

- jumlah baris dan jumlah kolom;
- nama kolom dan field yang diharapkan;
- tipe data seperti angka, teks, kategori, tanggal, atau boolean;
- preview beberapa baris;
- missing values yang muncul di tempat tidak terduga;
- masalah parsing, misalnya angka terbaca sebagai teks atau tanggal terbaca salah;
- duplikasi baris akibat join atau export;
- unit dan format yang tidak sesuai konteks project.

## Kenapa Ini Penting

Kesalahan saat memuat data bisa menyebar diam-diam. Pemisah desimal bisa membuat angka terbaca sebagai teks. Parser tanggal bisa menukar hari dan bulan. Join bisa menggandakan baris. Header bisa terbaca sebagai data.

Masalah seperti ini jauh lebih mudah ditangkap setelah data dimuat daripada setelah model terlanjur dilatih.
